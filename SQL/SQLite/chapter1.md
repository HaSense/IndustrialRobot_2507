# 1. 데이터란 무엇인가?

데이터(Data)는 컴퓨터가 저장하고 처리할 수 있는 모든 정보를 의미한다.

예)

| 생산일 | 제품명 | 생산수량 |
|----------|----------|----------|
| 2026-06-01 | Motor | 100 |
| 2026-06-01 | Sensor | 50 |
| 2026-06-02 | Motor | 120 |

위와 같은 생산 정보도 데이터이다.

---

# 2. 파일과 데이터베이스의 차이

초기 프로그램은 데이터를 파일에 저장하였다.

예)

- txt 파일
- csv 파일
- excel 파일

## 파일 방식

products.csv

```csv
id,name,qty
1,Motor,100
2,Sensor,50
3,Bearing,70
```

### 장점

- 구현이 쉽다.
- 사람이 직접 확인 가능하다.
- 별도 프로그램 설치가 필요 없다.

### 단점

- 데이터가 많아지면 검색이 느리다.
- 여러 사용자가 동시에 사용하기 어렵다.
- 데이터 중복이 발생한다.
- 데이터 무결성을 보장하기 어렵다.

---

## 데이터베이스 방식

데이터를 전문적으로 저장하고 관리하는 시스템이다.

예)

```sql
SELECT * FROM products;
```

### 장점

- 빠른 검색
- 데이터 중복 최소화
- 동시 사용자 처리
- 데이터 무결성 유지
- 백업 및 복구 지원

### 단점

- 학습이 필요함
- 초기 설정 필요

---

# 파일과 데이터베이스 비교

| 항목 | 파일 | 데이터베이스 |
|--------|--------|--------|
| 저장방식 | 텍스트 | 구조화된 저장 |
| 검색속도 | 느림 | 빠름 |
| 동시접근 | 어려움 | 가능 |
| 데이터중복 | 많음 | 적음 |
| 무결성 | 낮음 | 높음 |
| 관리기능 | 없음 | 풍부함 |

# 3. CSV의 한계

CSV(Comma Separated Values)는 데이터를 쉼표로 구분하는 텍스트 파일이다.

예)

```csv
date,product,qty
2026-06-01,Motor,100
2026-06-01,Sensor,50
2026-06-02,Motor,120
```

---

## 한계 1 : 검색이 느리다

Motor 제품만 찾으려면?

```c
while(fgets(line,sizeof(line),fp))
{
    ...
}
```

모든 행을 처음부터 끝까지 읽어야 한다.

데이터가 10건이면 괜찮다.

하지만

- 1만건
- 10만건
- 100만건

으로 증가하면 성능이 급격히 저하된다.

---

## 한계 2 : 데이터 중복

```csv
1,Motor,100
2,Motor,120
3,Motor,90
```

제품명이 계속 반복된다.

데이터가 커질수록 저장 공간이 낭비된다.

---

## 한계 3 : 데이터 오류

```csv
Motor
motor
MOTOR
Motro
```

모두 같은 제품인데 서로 다른 데이터로 저장될 수 있다.

---

## 한계 4 : 동시 수정 문제

사용자 A

```text
Motor 수량 수정
```

사용자 B

```text
Motor 수량 수정
```

같은 파일을 동시에 수정하면 충돌이 발생할 수 있다.

---

## 한계 5 : 관계 표현이 어렵다

생산관리 시스템 예시

제품 정보

```text
Motor
Sensor
Bearing
```

생산 이력

```text
2026-06-01 Motor 100
2026-06-02 Motor 120
```

CSV는 데이터 간 관계를 관리하기 어렵다.

---

# 4. DBMS란?

DBMS(Database Management System)

데이터베이스를 관리하는 소프트웨어이다.

사용자는 직접 파일을 다루지 않고 DBMS를 통해 데이터를 저장한다.

---

## DBMS의 역할

### 데이터 저장

```sql
INSERT INTO products VALUES(1,'Motor',100);
```

---

### 데이터 검색

```sql
SELECT * FROM products;
```

---

### 데이터 수정

```sql
UPDATE products
SET qty=200
WHERE id=1;
```

---

### 데이터 삭제

```sql
DELETE FROM products
WHERE id=1;
```

---

# 대표적인 DBMS

| 제품 | 특징 |
|--------|--------|
| SQLite | 가볍고 설치가 쉬움 |
| MySQL | 웹서비스에서 많이 사용 |
| MariaDB | MySQL 기반 오픈소스 |
| PostgreSQL | 강력한 기능 제공 |
| Oracle | 대규모 기업용 |
| SQL Server | Microsoft 제품 |

---

# 5. 관계형 데이터베이스(RDBMS)

RDBMS

Relational Database Management System

데이터를 테이블(Table) 형태로 저장한다.

---

## 제품 테이블

| id | name |
|----|----|
| 1 | Motor |
| 2 | Sensor |

---

## 생산 테이블

| id | product_id | qty |
|----|----|----|
| 1 | 1 | 100 |
| 2 | 1 | 120 |
| 3 | 2 | 50 |

---

### 관계(Relation)

```text
Motor (id=1)
      |
      +---- 생산이력
```

하나의 제품이 여러 생산 이력을 가질 수 있다.

이것이 관계형 데이터베이스의 핵심 개념이다.

---

# SQLite를 배우는 이유

SQLite는 다음과 같은 장점이 있다.

- 무료
- 설치가 거의 필요 없음
- 파일 하나로 관리
- C/C++ 연동 쉬움
- Python 연동 쉬움
- 임베디드 장비 사용 가능
- 스마트팩토리 소규모 시스템 구축 가능

예)

```text
factory.db
```

파일 하나가 데이터베이스이다.

# 핵심 정리

- CSV는 간단하지만 규모가 커지면 한계가 있다.
- 데이터베이스는 데이터를 체계적으로 관리한다.
- DBMS는 데이터 저장, 검색, 수정, 삭제를 담당한다.
- RDBMS는 테이블 간 관계를 관리한다.
- SQLite는 가볍고 배우기 쉬운 데이터베이스이다.
- 스마트팩토리, MES, 생산관리 시스템에서 데이터베이스는 필수 기술이다.
